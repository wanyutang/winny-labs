- Bullet Journal，由 Ryder Carroll 創立，核心概念是「快速記錄（Rapid Logging）」和「可自訂化的模組化管理」
- 兼具日誌、待辦、筆記、目標追蹤等功能，但形式非常自由，可以完全依照個人需求設計
- 系統自由，可手寫或數位化
- 將任務、筆記與日誌整合在同一本筆記中
- 強調整理思路、追蹤目標與習慣，而非單純列待辦

## Rapid Logging - 快速記錄

- • 代表行動 Action
	- 使用 Action 而非 Task，降低壓力，並非每個 Action 都需要立即執行
    - 在專案管理中，可定期篩選需要進入 Task 系統的 Action 進行追蹤
    - 可與其他待辦或 Agile 任務管理系統搭配使用
    - 狀態變化
        - • 初始化，表示可執行的下一步行動
        - / 處理中，表示行動正在執行
        - x 完成，表示行動已完成
- o 事件 Even
- - 筆記 Note
- = 心情 Moods
- / 處理中
- 其他標記
	- >  延期
	- >> 移至其他應用程式（如 Calendar、Todo、看板等）
	- < 搬到未來
	-  << 已設定排程提醒

Action 用法為初始化以 • 代表這是可執行的下一步行動，執行中時可以把 • 改為 / 以示意該行動正在執行，待完成後再直接改成 x

## 模組化頁面

- Future Log：長期規劃，用於記錄未來幾個月的重要事件、目標或待辦事項
- Monthly Log：月度概覽，通常包含月曆與本月任務清單
- Weekly / Daily Log：週或日記錄，列出每日任務、事件與筆記
- Collections：主題集合頁，用於追蹤特定主題或專案，例如習慣追蹤、閱讀清單、專案筆記等
- Key Page：關鍵頁面，用來記錄重要符號、快捷鍵、標記系統或常用模板
- Index：索引頁，用於快速定位筆記內容和頁面
- Intention Page：目標或意圖頁，記錄本期（如月、季、年）主要目標、方向或核心意圖
